# ClickFlow

Windows 桌面自動化工具 — 錄製或手動建立滑鼠、鍵盤、影像辨識操作序列，一鍵重複執行。

![Platform](https://img.shields.io/badge/platform-Windows%2010%2F11-blue)
![Python](https://img.shields.io/badge/python-3.11%2B-blue)
![License](https://img.shields.io/badge/license-MIT-green)

---

## 功能特色

| 功能 | 說明 |
|------|------|
| **多種動作類型** | 點擊、雙擊、右鍵、游標移動、等待延遲、鍵盤輸入、快捷鍵、**影像辨識點擊**、拖曳 |
| **錄製操作** | 直接在螢幕上操作，自動記錄每個步驟；雙擊自動合併、連續輸入自動合併 |
| **影像辨識** | 截取目標畫面區域，執行時自動在螢幕上找到並點擊，適合按鈕位置會變動的情境 |
| **鎖定視窗** | 執行前綁定目標視窗，視窗**移動或縮放**後自動補償所有座標，自動化與轉珠均支援 |
| **即時座標追蹤** | 視窗頂部即時顯示游標位置，按 `S` 鍵一鍵填入座標 |
| **彈性執行** | 設定執行輪數或無限循環，置頂迷你視窗顯示即時進度，`Space` 隨時中斷 |
| **錄製監視器** | 錄製中顯示置頂迷你視窗，即時列出每個已錄製步驟 |
| **多設定檔管理** | 儲存多組操作腳本，隨時載入切換 |


## 使用方式

### 錄製操作

1. 在右側「控制台」點擊 **● 開始錄製**
2. 切換到目標視窗，正常操作（點擊、輸入文字、按快捷鍵）
3. ClickFlow 視窗與錄製監視器上的點擊會自動排除，不會被記錄
4. 按 `F9` 或點擊 **■ 停止錄製** 結束
5. 步驟自動填入序列，可直接執行或手動微調

**錄製選項：**

| 選項 | 說明 |
|------|------|
| 錄製移動 | 額外記錄游標移動軌跡（預設關閉） |
| 延遲上限 | 停頓超過此秒數時壓縮為上限值（預設 5 秒） |

> 連續輸入的文字自動合併為一個步驟；兩次快速點擊同一位置自動識別為雙擊。

---

### 手動建立步驟

1. 左側「動作類型」選擇要執行的動作
2. 填入座標（或按 `S` 鍵自動填入目前游標位置）
3. 設定執行次數、延遲秒數等參數
4. 按 **＋ 新增步驟** 加入序列
5. 在步驟列表上**雙擊**可回到編輯器修改

---

### 影像辨識點擊

適合按鈕位置不固定、或每次啟動介面佈局略有不同的情境。

1. 動作類型選 **image_click**
2. 按 **📷 截取區域** — 螢幕變暗後，拖曳框住目標（例如某個按鈕或圖示）
3. 設定**相似度**（0.1–1.0，預設 0.85，越高要求越精確）和**超時秒數**（找不到則報錯）
4. 按 **＋ 新增步驟**

執行時程式自動在螢幕上搜尋該圖片，找到後點擊其中心點。截取的圖片儲存於 `images/` 資料夾。

---

### 執行

1. 右側「執行控制」設定輪數（勾選「無限循環」持續執行）
2. 按 **▶ 開始**，主視窗隱藏，右下角顯示置頂迷你監視器
3. 按 `Space` 或迷你視窗的 **■ 停止** 中斷

---

### 儲存與管理

- 右側「設定檔」區輸入名稱，按 **儲存設定檔** 儲存目前序列
- 「已儲存操作」列表點選後按 **載入**，或直接雙擊載入
- 按 **＋ 新增** 清空序列，開始全新腳本

## 🔮 轉珠（摩靈 / 神魔之塔類遊戲）

針對 Android 模擬器上的消珠類遊戲，自動截圖辨識盤面、計算最佳路線，並模擬手指拖曳完成轉珠。

> 僅支援 Windows 上的 PC 模擬器（BlueStacks、MuMu、LDPlayer 等），不支援直連手機。

### 初始設定（首次使用）

1. 切換到「🔮 轉珠」分頁
2. 選擇性：「綁定視窗」下拉選取模擬器視窗（模擬器移動或縮放後自動補償座標與格子尺寸）
3. 點擊 **📷 框選盤面** — 螢幕變暗後，拖曳框住整個珠盤區域（含邊緣格子）
4. 確認「列數」與「欄數」與遊戲一致（預設 5×6）
5. 點擊 **🔍 辨識測試** — 預覽區以顏色填色顯示辨識結果，確認各珠色正確

校準資料自動儲存，下次啟動無需重設。

### 執行轉珠

- 按 **▶ 執行轉珠** 或在遊戲中直接按 `F8`（不需切回 ClickFlow 視窗）
- 程式自動完成：截圖 → 辨識珠色（火水木光暗心）→ Beam Search 求解最佳路線 → 模擬拖曳

需在回合開始、珠盤靜止時手動觸發；遊戲技能特效期間可能辨識錯誤，等特效結束後再按。

### 連續模式

勾選「連續模式」並設定間隔秒數，轉珠完成後自動等待指定時間再重新偵測並執行，適合連續刷關。  
偵測到關卡結束（盤面空白超過半數或全格同色）時自動停止，或按 **■ 停止連續** 手動中止。

### 參數說明

| 參數 | 說明 | 預設 |
|------|------|------|
| 列數 / 欄數 | 盤面格子數 | 5 × 6 |
| 拖曳速度 | 每格移動耗時（毫秒），越小越快 | 25 ms |
| 求解精度 | Beam Search 寬度，越高路線品質越好但越慢 | 50（高精度） |
| 最大步數 | 路線長度上限 | 50 |

快速切換：**標準**（精度 30 / 步數 40）或 **高精度**（精度 50 / 步數 50，預設）。

### 限制

- 不支援特殊珠（鎖珠、毒珠、強化珠等），一律辨識為普通珠色
- 執行中移動或縮放模擬器視窗仍可能有一幀誤差；建議搭配「綁定視窗」，在轉珠開始前視窗保持靜止
- 需使用者手動觸發，不自動偵測回合開始

## 鎖定視窗的運作原理

「綁定視窗」功能同時追蹤視窗的**位置**與**尺寸**，**自動化（Tab 1）與轉珠（Tab 2）均支援**，視窗移動或縮放後無需重新錄製或重新校準。

### 基準記錄時機

| Tab | 記錄時機 |
|-----|---------|
| 自動化 | 從「綁定視窗」下拉選取視窗時 |
| 轉珠 | 從下拉選取視窗時；**或**完成「框選盤面」時（以校準當下為準） |

記錄的內容（透過 Win32 `GetWindowRect`）：
- **參考位置** `ref = (x, y)` — 視窗左上角螢幕座標
- **參考尺寸** `ref_size = (w, h)` — 視窗寬高（像素）

### 執行時的換算公式

每次執行前重新查詢視窗目前位置與尺寸，計算縮放比與位移：

```
sx = 目前寬 / 參考寬
sy = 目前高 / 參考高
```

**Tab 1 — 自動化步驟座標**（click、double\_click、right\_click、move、drag 起終點）：

```
new_x = 目前視窗 x + (step_x − ref_x) × sx
new_y = 目前視窗 y + (step_y − ref_y) × sy
```

**Tab 2 — 轉珠盤面座標**（board\_x/y 與格子尺寸）：

```
new_board_x = 目前視窗 x + (board_x − ref_x) × sx
new_board_y = 目前視窗 y + (board_y − ref_y) × sy
new_cell_w  = cell_w × sx
new_cell_h  = cell_h × sy
```

換算結果僅用於當次執行，原始錄製座標與校準設定值不受影響。

### 視窗消失再重開的處理

程式優先用 `hwnd`（視窗代號）查詢。若 `hwnd` 失效（視窗被關掉重啟），自動改用**視窗標題**重新掃描 `EnumWindows`，找到後更新快取，不需使用者重新選取。

## 熱鍵

| 鍵 | 時機 | 功能 |
|----|------|------|
| `S` | 任何時候 | 擷取目前游標座標，填入步驟編輯器 |
| `Space` | 執行中 | 立即停止執行 |
| `F9` | 錄製中 | 停止錄製 |
| `F8` | 轉珠模式中 | 觸發截圖→辨識→求解→拖曳完整流程 |

---

## 支援的動作類型

| 類型 | 說明 |
|------|------|
| `click` | 滑鼠左鍵點擊指定座標 |
| `double_click` | 滑鼠左鍵雙擊 |
| `right_click` | 滑鼠右鍵點擊 |
| `move` | 移動游標到指定座標（不點擊） |
| `delay` | 等待指定秒數 |
| `keyboard_input` | 輸入文字 |
| `hotkey` | 觸發快捷鍵（如 `ctrl+c`、`alt+F4`） |
| `image_click` | 在螢幕上找到參考圖片後點擊 |
| `drag` | 從起點拖曳到終點（mouseDown → moveTo → mouseUp） |

## 打包為執行檔

```bash
pip install pyinstaller
python -m PyInstaller ClickFlow.spec --noconfirm
```

輸出位於 `dist/ClickFlow.exe`，無需安裝 Python 即可執行。  
`clicker.db` 和 `images/` 均於首次執行時自動建立，不需預先準備。

## 專案結構

```
ClickFlow/
├── main.py                  # 入口：logging → DB → GUI
├── models/
│   ├── click_step.py        # 步驟資料類別
│   ├── profile.py           # 設定檔資料類別
│   └── orb_config.py        # 轉珠盤面校準設定
├── services/
│   ├── database_manager.py  # SQLite CRUD
│   ├── click_executor.py    # 執行執行緒（click/drag/image_click 等）
│   ├── recorder.py          # pynput 全域錄製服務
│   ├── keyboard_monitor.py  # 全域熱鍵（Space/S/F8/F9）
│   ├── mouse_tracker.py     # 即時座標追蹤
│   ├── window_manager.py    # Win32 視窗列舉與位置查詢（鎖定視窗用）
│   ├── orb_board.py         # 截圖 → 盤面辨識（HSV 色相比對）
│   ├── orb_solver.py        # Beam Search 轉珠路線求解
│   └── orb_executor.py      # 路線轉換為滑鼠拖曳執行緒
├── views/
│   ├── main_window.py       # 主視窗（Tab1 自動化 / Tab2 轉珠）
│   └── orb_calibrate.py     # 轉珠盤面校準視窗
├── docs/
│   ├── orb_solver_spec.md   # 轉珠功能完整開發規範
│   └── dev_log.md           # 開發決策與踩坑記錄
└── images/                  # 影像辨識參考圖片（執行時自動建立）
```
